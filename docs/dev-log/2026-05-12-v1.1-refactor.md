## 摘要

本次对 auto-debug Skill 进行了大规模架构重构，相当于从 v1.0 的 monolithic 设计演进为 v1.1 的模块化、渐进式披露架构。核心驱动力是：减少无关上下文污染、支持零配置项目适配扩展、实现零项目污染、以及统一规范化的 skill 分发方式。

重构后 SKILL.md 从 728 行精简为 435 行，项目特定知识被提取到按需加载的 references/ 文件中，新增项目支持无需再改动 SKILL.md 本身。

## 变更清单

### 架构层面

| 变更 | 之前 | 之后 |
|------|------|------|
| **Skill 结构** | 单文件 monolithic（728 行） | 通用核心（435 行）+ 按需加载 references/ |
| **项目适配** | 硬编码在 SKILL.md prompt 中 | `config/project-adapters-config.json` 注册表 + references/*.md |
| **配置来源** | 项目级 `.auto-debug-config.json` + 全局 `~/.auto-debug/config.json` + 内置默认值 | **唯一来源**：skill 内置 `config/default-config.json` |
| **上下文加载** | 一次性全量加载所有项目知识 | **渐进式披露**：识别项目类型后才加载对应 reference |
| **输出路径** | 直接写在日志根目录 | 统一写在 `{log_path}/auto-debug/` 或 `%TEMP%/auto-debug/` |
| **安装方式** | 手动复制 SKILL.md | `npx skills add owner/repo --skill auto-debug` |
